8d. Freeze the rarity table after the dev-mint window

v11 ships Brawlers.freezeRarity(): once the dev wallet has taken its mints, this owner-only call locks the rarity table so subsequent buyers see no further skip-rare swaps. The shuffle is deterministic from masterSeed, so anyone can re-derive the post-shuffle hash off-chain and confirm initialRarityHash matches.

# Verify the on-chain table still matches the canonical shuffle:
cast call $BRAWLERS_ADDRESS "initialRarityHash()(bytes32)" --rpc-url $MAINNET_RPC
cast call $BRAWLERS_ADDRESS "rarityHash()(bytes32)"        --rpc-url $MAINNET_RPC
# If they differ, dev-skip swaps have fired (expected if the dev wallet has minted).

# Lock it in (idempotent):
cast send $BRAWLERS_ADDRESS "freezeRarity()" \
  --private-key $DEPLOYER_KEY --rpc-url $MAINNET_RPC

After this tx, public buyers can be sure the rarity they pull is exactly what's at their tokenId in the now-frozen table. Call this before posting the rarity sheet to Discord/X.

16. Renounce sequence (24-48h after launch)

After trading has stabilised + obvious snipers blacklisted, surrender control of BRAWL:

16a. Lift the launch limits

cast send $BRAWL_ADDRESS "liftLimits()" \
  --private-key $DEPLOYER_KEY --rpc-url $MAINNET_RPC

One-way. Removes max-tx and max-wallet caps. Trading is now unrestricted.

16b. Renounce BRAWL ownership

cast send $BRAWL_ADDRESS "renounceOwnership()" \
  --private-key $DEPLOYER_KEY --rpc-url $MAINNET_RPC

Whitelist/blacklist/limits frozen forever. BRAWL contract becomes truly immutable.

Game contracts (Duel/MintDrop/Marketplace/Graveyard) STAY dev-controlled for tuning. No rug-able functions in those — only economics setters.
